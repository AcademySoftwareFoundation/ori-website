# Contributing

Thanks for helping improve the Open Review Initiative website!

## Making a change

1. Fork the repository (or create a branch if you have write access).
2. Make your edits — see the [README](README.md) for where content lives.
3. Preview locally with `hugo server` and confirm the affected pages render correctly.
4. Open a pull request against `main`. The CI build must pass before merge.

## Contributor License Agreement

This repository uses the Linux Foundation [EasyCLA](https://easycla.lfx.linuxfoundation.org/)
check. Make sure the email on your commits (`git config user.email`) matches the email
registered with your CLA, or the check will block the pull request.

## Questions

ORI discussion happens in the ASWF [Professional Review Working Group](https://lf-aswf.atlassian.net/wiki/spaces/PRWG/overview).
