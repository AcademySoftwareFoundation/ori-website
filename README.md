# Open Review Initiative Website

Source for [openreviewinitiative.org](https://openreviewinitiative.org/), the website of the
[Academy Software Foundation](https://www.aswf.io) (ASWF) Open Review Initiative (ORI).

The site is built with [Hugo](https://gohugo.io/) using the
[hugo-universal-theme](https://github.com/devcows/hugo-universal-theme), and deployed to
GitHub Pages automatically on every push to `main`.

## Repository layout

| Path | Purpose |
|---|---|
| `hugo.toml` | Site configuration: menus, footer copyright, section toggles |
| `content/_index.md` | Front-page content sections |
| `data/features/*.yaml` | The feature tiles (OpenRV, xSTUDIO, RPA, encoding guidelines, …) |
| `data/carousel/*.yaml` | Carousel slides |
| `layouts/` | Site-level template overrides (carousel, video, cookie-consent script) |
| `static/` | Site assets served as-is (`img/` logos and media, `css/` custom styles) |
| `themes/hugo-universal-theme/` | Vendored theme (do not edit; override via `layouts/`) |
| `.github/workflows/hugo.yml` | CI: builds the site and deploys `public/` to GitHub Pages |

## Editing the site

Most content changes need no HTML:

- **Feature tiles** — edit or add a YAML file in `data/features/`. Each file has a
  `name`, `icon`, `url`, `description`, and `weight` (display order).
- **Front-page text** — edit `content/_index.md`.
- **Menus, footer, section toggles** — edit `hugo.toml`.
- **Images / video** — add files under `static/img/` and reference them as `/img/<name>`.

To change page structure, copy the relevant template from
`themes/hugo-universal-theme/layouts/` into `layouts/` and edit the copy — Hugo prefers
site-level templates over theme templates of the same name.

## Previewing locally

Install [Hugo extended](https://gohugo.io/installation/) (the site is authored against
v0.147.8), then:

```sh
hugo server
```

and open <http://localhost:1313/>. The build output directory (`public/`) is generated by
CI and must not be committed.

## Deployment

Merging to `main` triggers `.github/workflows/hugo.yml`, which builds the site with the
production `baseURL` (`https://openreviewinitiative.org/`) and publishes it to GitHub
Pages. Pull requests run the build step only, as a validation check.
